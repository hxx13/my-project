与你「第三步：Windows 启动 frpc、本机 Spring」的对齐检查清单
================================================================

1) 本机 Spring
   - 默认监听端口见项目 application.properties 的 server.port（当前仓库为 8080，与 frpc.toml 的 localPort 一致）。
   - 公网仍通过 frps 的 remotePort（如 8080）暴露；云函数 SPRING_BASE_URL 写公网 IP:8080 时，与 remotePort 对应，不必等于本机 server.port。

2) 客户端配置（Windows）
   - 客户端模板仅此一份：本目录（cloud-relay）下的 frpc.toml。
     frpc.exe 若在 deploy\frp_0.56.0_windows_amd64，请复制该文件到 exe 同目录（见该目录 README.txt），勿在多处各维护一份。
   - 启动：frpc.exe -c frpc.toml
   - 成功时日志中应出现类似：
       [springboot-proxy] start proxy success

3) 服务端 frps（阿里云 ECS）必须与 frpc.toml 一致
   - bind_port = 7000（与 serverPort 一致）
   - token 与 frpc.toml 里 auth.token 完全一致（INI 用 token=，TOML 用 auth.token）
   - 若 frpc 已能启动但报「token in login doesn't match token from configuration」：仅 token 不一致，以 ECS 上 frps 为准，把本机 frpc.toml 的 auth.token 改成与服务器逐字相同（勿多空格/勿混用另一份模板里的占位符）。
   - 【关键】公网要访问 47.101.61.184:8080 时：
       frps 中 proxy_bind_addr 须为 0.0.0.0
     若为 127.0.0.1，则代理端口只绑在 ECS 回环地址上，外网永远连不上 8080。

4) 安全组（阿里云控制台）
   - 放行 TCP 7000（frp 控制通道）
   - 放行 TCP 8080（Spring 穿透业务口）
   - 勿与宝塔其它服务端口冲突；若 8080 已被占用，可改 frpc.toml 的 remotePort 与 frps 放行端口一致。

5) 与「宝塔 + Nginx 旧方案」的关系
   - frpc.ini 使用 remote_port=18080 + frps 仅 127.0.0.1 监听 + Nginx 443 反代到 18080：与「公网 8080」是两套架构，二选一。
   - 云函数 SPRING_BASE_URL 若写 http://47.101.61.184:8080，对应的是「公网 8080 + frpc.toml」这套。

6) frp 版本
   - frpc 与 frps 主版本号一致；TOML 为新配置格式，过旧 frp 可能只认 INI。

7) 若 frpc 启动即报 error unmarshaling JSON / cannot unmarshal string into v1.ClientConfig
   - 最常见：文件头含 UTF-8 BOM（十六进制 EF BB BF）。用十六进制查看首字节应为 23（#）或 73（s），不能是 EF。VS Code 右下角选「UTF-8」勿选「UTF-8 with BOM」后保存。
   - 检查 TOML：auth.method、auth.token、serverAddr 等字符串是否带英文双引号，勿写 token=xxx 旧 INI 顶层。
   - 保存为 UTF-8（无 BOM）；勿用中文弯引号。
   - 若有 featureGates，须写在一行：featureGates = { VirtualNet = true }，勿用多行 { } 块。
