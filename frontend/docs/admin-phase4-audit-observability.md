# 阶段四：审计与可观测（规划边界，未实现）

本文件为后续迭代的接口与责任边界说明，与当前已交付的「导航统一 + 命令面板 + 表格壳」解耦。

## 审计（Audit）

- **目标**：对管理端写操作记录「操作者、时间、资源标识、动作、摘要/前后差异（视脱敏策略）」。
- **后端**：Spring 侧建议统一入口（如 `@ControllerAdvice` + 注解标记、或 AOP 切面）写入审计表；需 DDL 时遵守仓库 `.cursor/rules/db-schema-ddl-mandatory.mdc`（`schema.sql` + `scripts/*.ddl.sql`）。
- **前端**：只读「操作日志」列表页 + 筛选/导出（权限仅 SUPER_ADMIN 或独立审计角色，由业务定）。
- **与导航无关**：审计不改变 `adminNavRegistry` 或命令面板数据源。

## 可观测（Observability）

- **请求关联**：后端在响应头返回 `X-Request-Id`（或 trace id）；前端 axios 响应拦截器读取并挂到全局（如 `window.__lastRequestId`）或错误 toast 上下文，便于用户报障时对齐日志。
- **错误聚合**：生产环境可接入 Sentry（或同类）上报未捕获异常与关键路由错误边界。
- **性能**：关键大屏/管理列表可上报 Web Vitals（LCP、FID 等），与表格壳无强绑定。

## 安全与合规提示

- 导出、批量删除等高危操作应同时落审计与二次确认。
- 个人敏感字段展示与导出需按数据分级脱敏。
